Vue lifecycle

生命周期

  1. 总体分成：初始化 -> 运行 -> 销毁
  2. 详细：开始创建 -> 初始化数据 -> 编译模板 -> 挂载DOM 、渲染 -> 更新 -> 渲染、销毁
  3. 实例化组件new Vue()
  4. 初始化事件和生命周期：init events、init cycle
  5. beforeCreate:
    是new Vue()之后触发的第一个钩子，在当前阶段data、methods、computed以及watch上的数据和方法都不能被访问。数据观测(Data Observer)和(event/watcher事件)配置前调用
  6. created:
    在实例创建完成后被立即调用，当前阶段已经完成了数据观测Data Observer，也就是可以使用数据，DOM未生成，$el属性还不存在，即没有动态创建template属性对应的Html元素，在这里更改数据更改数据不会触发updated函数。所以可以做一些初始数据的获取，在当前阶段无法与Dom进行交互，如果非要想，可以通过vm.$nextTick来访问Dom。
  7. Has "el" option:
    • 检查el属性，检查Vue配置，如果发现配置项上有el选项，则自动调用$mount方法，也就是说有了el选项，就不需要再手动调用$mount方法，反之，没提供el选项则必须调用$mount
    • 检查template属性，配置中的template项，如果没有template,会把el作为被填充对象替代填充对象即;如果对象有template，那么template模板会替换$el对应得内容。
    • 检查render属性：render函数若存在，则Vue构造函数不会从template选项或通过el选项指定的挂载元素中提取出的HTML模板编译渲染函数。
  8. beforeMount:
    • 在挂载开始之前被调用：相关的render函数首次被调用
    • 发生在挂载之前，在这之前template模板已导入渲染函数编译。而当前阶段虚拟Dom已经创建完成，即将开始渲染。在此时也可以对数据进行更改，不会触发updated。此时this.$el有值。
  9. 模板编译：用vue对象的数据替代template里的内容
  10. mounted:
    • 在模板编译完成、数据挂载(把数据显示在模板里)完成后发生，在当前阶段，真实的Dom挂载完毕，数据完成双向绑定，可以访问到Dom节点，使用$refs属性对Dom进行操作。
    • mounted不会保证所有的子组件也都一起被挂载。如果你希望等到整个视图都渲染完毕，可以在mounted内部使用vm.$nextTick
  11. beforeUpdate:
    • 数据更新前时调用，也就是响应式数据发生更新，虚拟Dom重新渲染之前被触发
    • 适合在更新之前访问现有的 DOM，比如手动移除已添加的事件监听器。
    • 在当前阶段进行更改数据，不会造成重渲染。
  12. updated:
    • 数据更改导致的虚拟DOM重新渲染和打补丁(组件更新)之后调用的钩子
    • 要注意的是避免在此期间更改数据，因为这可能会导致无限循环的更新。
  13. beforeDestroy:
    • 实例销毁之前调用。在这一步，实例仍然完全可用。
    • 可以在这时进行善后收尾工作，比如清除计时器。
  14. destroyed:
    实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，数据绑定被卸除，所有的事件监听器被移除，所有的子实例也都被销毁。
  15. activated:
    被keep-alive缓存的组件激活时调用。
  16. deactivated:
    被keep-alive缓存的组件停用时调用。

父子组件的生命周期

组件的调用顺序都是先父后子,渲染完成的顺序是先子后父。

组件的销毁操作是先父后子，销毁完成的顺序是先子后父。

加载渲染过程

父beforeCreate->父created->父beforeMount->子beforeCreate->子created->子beforeMount->子mounted->父mounted

子组件更新过程

父beforeUpdate->子beforeUpdate->子updated->父updated

父组件更新过程

父 beforeUpdate -> 父 updated

销毁过程

父beforeDestroy->子beforeDestroy->子destroyed->父destroyed

提问

异步请求在哪一步发起？

可以在钩子函数 created、beforeMount、mounted 中进行异步请求，因为在这三个钩子函数中，data 已经创建，可以将服务端端返回的数据进行赋值。

如果异步请求不需要依赖 Dom 推荐在 created 钩子函数中调用异步请求，因为在 created 钩子函数中调用异步请求有以下优点：

  • 能更快获取到服务端数据，减少页面 loading 时间；
  • ssr 不支持 beforeMount 、mounted 钩子函数，所以放在 created 中有助于一致性;
