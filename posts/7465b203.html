Vue 数据响应式的实现原理

Object

JS中检测对象的变化有两种：Object.defineProperty和ES6的proxy

收集依赖

initState

src\core\instance\state.js

  • 数据响应式的入口：处理 props、methods、data、computed、watch
  • 优先级：props、methods、data、computed(其中 computed 中的 key 不能和 props、data 中的 key 重复，methods 不影响)

initProps

src\core\instance\state.js

  • 处理props，为变量设置响应式，并挂载至vm实例上

proxy

src\core\instance\state.js

  • 设置代理，将 key 代理到 target 上

initMethods

src\core\instance\state.js

  • 校验methods[key]类型为函数
  • 与props判重处理(还有vue内置函数)
  • 挂载至vm实例上(vm[key] = methods[key])

initData

src\core\instance\state.js

  • 判重处理
  • 挂载data对象属性到vm实例
  • 响应式处理

initComputed

src\core\instance\state.js

  • 为computed[key]创建watcher实例，默认懒执行
  • 挂载computed[key]到vm实例
  • 判重处理

initWatch

src\core\observer\index.js

  • 响应式处理的入口函数
  • 为对象创建观察者实例，如果对象已经被观察过，则返回已有的观察者实例，否则创建新的观察者实例

observe

src\core\observer\index.js

  • 响应式处理的入口函数
  • 为对象创建观察者实例，如果对象已经被观察过，则返回已有的观察者实例，否则创建新的观察者实例

Observe

src\core\observer\index.js

  • 观察者类，会被附加到每个被观察的对象上，value.ob = this
  • 对象的各个属性则会被转换成 getter/setter，并收集依赖和通知更新

defineReactive

src\core\observer\index.js

  • Object.defineProperty拦截obj[key]的读取和操作
  • 首次读取收集依赖和属性更新通知依赖更新

dependArray

src\core\observer\index.js

  • 遍历每个数组元素，递归处理数组项为对象的情况，为其添加依赖
  • 因为前面的递归阶段无法为数组中的对象元素添加依赖

Array Observer

src\core\observer\array.js

  • 遍历每个数组元素，递归处理数组项为对象的情况，为其添加依赖
  • 因为前面的递归阶段无法为数组中的对象元素添加依赖

def

src\core\util\lang.js

protoAugment

src\core\observer\index.js

  • 设置 target.proto 的原型对象为 src
  • 比如 数组对象，arr.proto = arrayMethods

copyAugment

src\core\observer\index.js

  • 在目标对象定义指定属性
  • 如把arrayMethods里的arrKeys全部赋给目标数组

Dep

src\core\observer\dep.js

提问

Vue2.0响应式原理是怎么实现的

响应式的核心是通过 Object.defineProperty 拦截对数据的访问和设置
响应式的数据分为两类：

  • 对象，循环遍历对象的所有属性，为每个属性设置 getter、setter，以达到拦截访问和设置的目的，如果属性值依旧为对象，则递归为属性值上的每个 key 设置 getter、setter访问数据时(obj.key)进行依赖收集，在 dep 中存储相关的 watcher设置数据时由 dep 通知相关的 watcher 去更新

  • 数组，增强数组的那 7 个可以更改自身的原型方法，然后拦截对这些方法的操作添加新数据时进行响应式处理，然后由 dep 通知 watcher 去更新删除数据时，也要由 dep 通知 watcher 去更新

  • 数组考虑性能原因没有用 defineProperty 对数组的每一项进行拦截，而是选择对 7 种数组（push,shift,pop,splice,unshift,sort,reverse）方法进行重写(AOP 切片思想)所以在 Vue 中修改数组的索引和长度是无法监控到的。

computed 和 watch 有什么区别

  • computed 和 watch 的本质是一样的，内部都是通过 Watcher 来实现的
  • computed 是计算属性，依赖其他属性计算值，并且 computed 的值有缓存，只有当计算值变化才会返回内容，它可以设置 getter 和 setter。watch 监听到值的变化就会执行回调，在回调中可以进行一些逻辑操作。一般用于同步计算
  • 侦听属性watch适用于当数据变化时执行异步或者开销较大的操作时使用，即需要完成一段复杂的业务逻辑可以放在 watch 中
